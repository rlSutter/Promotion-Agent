# Product

## Register

product

## Users

Independent Substack writers running their own promotion workflow. Solo operators, not teams. They publish thoughtfully and hate the 45-minute post-publish ritual of crafting platform-specific copy for LinkedIn, Substack Notes, and Bluesky, figuring out what to comment on in their ecosystem, and remembering what they've written before. They open this dashboard after publishing, make decisions, copy drafts, and close it. It's a focused 15-minute window, not an always-on tool.

## Product Purpose

An autonomous agent that watches the writer's RSS feed, extracts key sentences worth promoting, generates platform-native drafts, surfaces commenting opportunities, builds a searchable article inventory, and produces weekly on-ramp posts. Everything lands in the dashboard for human review, edit, and approval. The agent handles the systematic; the writer handles the judgment. Success means the writer publishes better-matched content across more platforms in less time, without it feeling like work.

## Brand Personality

Warm, editorial, human. Feels like a creative companion built for one person, not an enterprise SaaS product. Voice: precise but not cold, confident but not boastful. 3 words: calm, personal, clear. The emotional goal is the feeling of having a thoughtful collaborator who handles the logistics so the writer can stay in creative mode.

Reference point: Substack's own editor. Writer-native, calm, editorial without corporate formality. Content is what matters; the interface steps back.

## Anti-references

- Social media playfulness: rounded bubbly cards, emoji in every heading, consumer-app energy. The dashboard already uses emojis as section markers; these should be toned down or removed, not amplified.
- Generic SaaS: cream background, gradient hero stats, blue CTA buttons, shadow-card grids. The current dashboard is already too close to this; any redesign should move away from it.
- Enterprise density: sidebar nav crammed with icons, data-table-first layouts, corporate gray palette.

## Design Principles

1. **Content before chrome.** The promotional copy is the product. The interface is infrastructure. If a UI element isn't helping the writer evaluate or publish a draft, it has no reason to exist.
2. **Workflow, not performance.** Every decision — layout, spacing, interaction — exists to reduce friction in the review-edit-copy-publish loop. Nothing to impress; everything to accelerate.
3. **Writer voice.** The tool is an extension of the writer's creative process. It should feel like their notebook, not a SaaS dashboard. That means warmth in the typography, restraint in the decoration, and respect for the copy being reviewed.
4. **Calm clarity.** One section at a time, unambiguous state (pending, published, skipped), no urgency theater. The writer is already done with the hard work of publishing; this tool should feel like winding down, not ramping up.
5. **Personal scale.** Built for one person. Not scalable to a team, not designed to impress stakeholders. Intimate, not enterprise.

## Accessibility & Inclusion

WCAG AA compliance. Keyboard navigable. Sufficient color contrast for all interactive states. No color-only state communication (complement with text labels or icons). Support system preference for reduced motion.
